以太网数据帧

以太网的数据帧格式如下图所示

以太网帧的类型:

0x0800：IPv4 : 指示该数据帧按ipv4处理

0x0806：ARP : 用于ip地址到mac地址的转换，该以太帧一般出现在主机没有得到mac地址，向网络广播寻找mac的过程中

0x8100：VLAN Tag（TPID）: 拥有VLAN tag的帧，该帧旨在指示该帧添加了一个字段，用于指示数据帧所在的vlan

0x86DD：IPv6 : 指示该数据帧按ipv6处理

0x88F7：PTP / gPTP : 用于精确控制时间

0x22F0：AVTP

数据链路层一个重要的物件: 交换机

VLAN机制

在交换机之间传递过程中，需要使用VLAN tag的帧

位于不同VLAN内的主机之间 不能 直接通信，需要通过路由器或三层交换机等网络层设备进行转发。

VLAN接口是一种三层模式下的虚拟接口，主要用于实现VLAN间的三层互通，它不作为物理实体存在于交换机上。每个VLAN对应一个VLAN接口，该接口可以为本VLAN内端口收到的报文根据其目的IP地址在网络层进行转发。通常情况下，由于VLAN能够隔离广播域，因此每个VLAN也对应一个IP网段，VLAN接口将作为该网段的网关对需要跨网段转发的报文进行基于IP地址的三层转发。

SFP机制

简而言之，用于在交换机之中选举一个根交换机，允许该机上的所有端口通讯，其余交换机可以到达根交换机，但是有且仅有一条，通过阻塞端口实现

1. 段表设计GDT,LDT

一个32位系统需要一个GDT，可以同时拥有多个LDT,每一个LDT通过GDT来获取LDT所在的位置

在INTEL规范中，第一个GDT表项应该是不被使用的，当cpu试图通过该selector来读取内存时，会返回GP（general protection)
错误，该表项是希望系统初始化selector指向该表项，并在之后进行更改，如果没有更改，那么selector可以被视为未进行赋值，从而产生异常对计算机系统进行保护。

INTEL文档中对GDT的位置进行了建议:GDT的起始地址（base address)
对齐8字节边界（个人认为:BASE ADDRESS % 0x1000 = 0) 来增强cpu的处理能力，

同时，规定 BASE ADDRESS + LIMIT = GDT_END ADDRESS
从而指定GDT的范围大小。

值得注意的是，执行SGDT和SLDT时，应该让存储地址对齐，从而避免可能的用户模式下的对齐检查而出错

总体段式设计:

该段式设计的最终目的是从一个逻辑地址转换为线性地址

(如果开启分页，那么还需要从线性地址通过页表转化为物理地址，否则直接转换为物理地址)
